# 软考中级软件设计师

## 前言

昨天考了软考中级软件设计师，整体来说是一个感觉还可以，虽说上午题挺没信心的，考到了挺多没背到的东东，但感觉应该能过（总不至于我没把握的选择题全选错吧）

成绩好像要两个月才出来，那个时候我早就忘了感受了，所以虽然不知道过没过，还是来分享一下心得。

## 警告

**不建议没有计算机基础的同学报考软件设计师！**

对我们这一类计算机出身的，看这些知识的感觉虽然是 **啊！这玩意在大学里学过，我还给老师了！**
但是重新学起来是非常容易非常快的，对我来说最难的是那种要背要记很多零散知识点的选择题。

如果是没基础的同学，不说数据结构与算法这种（这玩意我当时学起来就很头痛了），单说数的表示（原码，反码，补码），或者面向对象UML这种简单题（对科班出身而言这些都是是基础题）恐怕都能让人头大。 **很多我们觉得是常识的内容在没学过计算机的人看来就是天书（正则表达式，数据结构与算法，UML，设计模式，操作系统知识等）**

## 体感

我出考场的时候听到有人抱怨大题很难。 **对我来说大题比选择题那种一堆要背的简单多了**，照着UML给JAVA代码填空不是送分吗？把子类的函数签名抄到父类上，给一个成员变量补充它的类型，填空就仅仅是这样的难度…画ER图不是送分吗？对着需求写UML中的类名，用例名，或者照着数据流图补充实体名考的也只是细心程度。这次考的算法代码填空题也不是DP而是回溯法，用的是递归实现，还贴心地给你写出了剪枝思路和代码注释…

最怕的还是选择题零零散散的要死记硬背的知识点……我数了下大概有25+题没把握！要知道选择题只能错30题！

软考是那种及格就行的考试，75选择题，5道大题。 **注意，选择题和大题都要及格才行，不是算总分的。选择题提前交卷，多出来的时间可以给大题。**

其中大题有一道面向对象代码填空是C++和java二选一，虽然我是C++出身，但我还是选了java作答，要是什么接口忘了写成纯虚函数，或者不小心把箭头写成点就太冤枉了。

## 准备策略

我一开始傻傻地看官方出的那个教程……浪费了很多时间，那本写写得不好，很多地方不好理解，然后不管是重点还是非重点都写得密密麻麻，看起来很花时间。

**建议还是看完了重点知识，做了一些题目之后，再对这本书补漏。**

看完了重点知识之后就可以开始做历年题目了。

后来我没什么时间就看上面分类讲解知识的博客了，然后这个月越临近考试越是开摆。这种心态不可取，我这两个星期恐怕加起来只看了不到一天知识，做了不到一天题……如果多做一些选择题，多看一下知识点，大概对通过也会有十足的信心了。

大题的话没什么好讲的，多做一下历年试题你心里就有数了，大概知道怎么做了。选择题……多背多记吧……

**上面那个上午题和下午题的博客写得非常好，强烈推荐看看。**

**注意官网上的时间还有模拟练习的时间！一定要用一下那个模拟考试系统，熟悉一下机考形式！**

## 心态

不用紧张，你看我，两个月有一月在开摆，历年题没做几道（只做了23年和22年的），就算这样，选择题也只是二十多题没把握，要知道你是可以错三十题的（其实还是有点心虚的，要是真错了很多，剩下有把握的题里也有错的，那我就爬了）

如果你认真准备，肯定是能顺利通过的。
